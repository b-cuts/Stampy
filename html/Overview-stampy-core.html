Stampy, a Java implementation of the STOMP 1.2 specification

Release $version, $date

stampy-core-$version

This library provides class representations of STOMP 1.2 messages and the ability to convert to and from the classes. It can be used with the additional Stampy libraries or it can be used with existing STOMP client or server implementations.

All such classes implement the StampyMessage interface

This library provides representations of STOMP 1.2 messages and a structure for implementing clients and servers. A reference implementation using Apache MINA is provided. The following artifacts are available from the [insert repo name here]:

  • stampy-core - the core Stampy classes
  • stampy-MINA-client-server-RI - the MINA reference implementation, includes stampy-core
  • stampy-examples - example code, includes stampy-MINA-client-server-RI
  • stampy-all - the whole Stampy project

Design considerations

  • Ease of configuration and use
  • DI framework-friendly
  • Minimal dependencies
  • Massively multithreaded, many clients can communicate with a server simultaneously
  • Fast (11 microseconds per message during testing)
  • Reliable
  • Extensible
  • Security focused

stampy-MINA-client-server-RI

Dependencies

  • <dependency org="asia.stampy" name="stampy-core" rev="$version"/>
  • <dependency org="org.apache.mina" name="mina-core" rev="2.0.7"/>

Description

The MINA RI provides a STOMP 1.2 client and server implementation. By themselves they are not fully STOMP 1.2 compliant. Compliancy and configuration are obtained by adding StampyMessageListener implementations to the client and server instances. Note that the order of adding the listeners to the gateways is important. The following reference implementations are included in the distribution:

SecurityMessageListener

At least one user implemented version of this sub interface of StampyMessageListener must be added and one must be first in the list, else a security exception is thrown. Implementations are to provide adequate security checks for their system.

MINA Client StampyMessageListeners

  • MinaConnectedMessageListener
  • MinaDisconnectListenerAndInterceptor

MINA Server StampyMessageListeners

  • MinaConnectListener
  • MinaConnectResponseListener
  • MinaHeartbeatListener
  • MinaLoginMessageListener
  • MinaAcknowledgementListenerAndInterceptor
  • MinaReceiptListener
  • MinaTransactionListener
  • VersionListener

Load Test

The load test client and server serve to demonstrate the efficiency of the MINA RI. By default one million ACK messages are generated by the client and each ACK causes the server to generate a RECEIPT message. Configuration of the client and server is done programmatically. Listeners for the client and server determine behaviour.

Log4j Level Changer

This example demonstrates the ability to send a Java binary from a client for exectution on the server. For demonstration purposes only.

System

The system example demonstrates how to configure a STOMP 1.2 compliant client and server using the StampyMinaMessageListener RIs listed above. The client and server exercise these implementations to determine compliancy.
